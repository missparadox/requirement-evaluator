# 中文报告模板

按下面结构输出中文 Markdown 报告。

## 1. 评估概览

- 数据源
- 需求条目数
- 评分方法
- 总体平均分
- 等级分布
- 是否达到“优秀设计标准”的总体结论

## 2. 总体发现

- 最常见优点
- 最常见问题
- 最薄弱的维度
- 最值得优先修补的结构性问题

## 3. 单条需求评估结果

每条需求至少包含：

- 需求标识与名称
- 总分与等级
- 各维度分数
- 关键证据
- 红旗问题
- 缺失项
- 修改建议

建议格式：

```md
### 需求 N: <编号> <名称>

- 总分: 78/100
- 等级: B
- 结论: 设计基础可用，但还未达到优秀设计标准

| 维度 | 分数 | 说明 |
| --- | ---: | --- |
| OR-用户语言描述 | 6/10 | ... |

关键证据：
- ...

红旗问题：
- ...

缺失项：
- ...

修改建议：
1. ...
2. ...
3. ...
```

## 4. 优先级建议

按高、中、低列出跨需求的改进动作。

## 5. 附录

- 维度定义摘要
- 评分边界说明
