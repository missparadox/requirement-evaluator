# 中文报告模板

按下面结构输出中文 Markdown 报告。

## 1. 评估概览

- 数据源
- `sheet_name`（如果输入是 Excel）
- OR 条目数
- DR 条目数
- 评分方法
- 总体平均分
- 等级分布
- 是否达到“优秀设计标准”的总体结论

## 2. 总体发现

- 最常见优点
- 最常见问题
- 最薄弱的维度
- 最值得优先修补的结构性问题

## 3. 单个 OR 评估结果

```md
### OR N: <编号> <名称>

- OR总得分: <分数>/100
- 等级: <等级>
- 结论: <结论>
- OR部分得分: <分数>/40
- DR得分:
  - DR-1 `<编号> <名称>`: <分数>/40
  - DR-2 `<编号> <名称>`: <分数>/40
- DR平均分: <分数>/40
- 需求分解与追踪质量得分: <分数>/20

| 部分/维度 | 分数 | 说明 |
| --- | ---: | --- |
| OR-用户语言描述 | <分数>/12 | <说明> |
| OR-应用场景 | <分数>/12 | <说明> |
| OR-用户价值 | <分数>/10 | <说明> |
| OR-约束和限制 | <分数>/6 | <说明> |
| DR-1总分 | <分数>/40 | <说明> |
| DR-2总分 | <分数>/40 | <说明> |
| 需求分解与追踪-范围与边界 | <分数>/6 | <说明> |
| 需求分解与追踪-假设与依赖 | <分数>/6 | <说明> |
| 需求分解与追踪-一致性与可追踪性 | <分数>/4 | <说明> |
| 需求分解与追踪-异常与边界场景 | <分数>/4 | <说明> |

关键证据：
- <证据1>
- <证据2>

扣分依据：
- <若命中红线规则，写明规则编号与分数上限依据>

红旗问题：
- <问题1>

缺失项：
- <缺失1>

修改建议：
1. <建议1>
2. <建议2>
3. <建议3>
```

## 4. 优先级建议

按高、中、低列出跨 OR 的改进动作。

## 5. 附录

- 维度定义摘要
- 评分边界说明
- 分数计算说明：
  - 单个 OR 总分 = OR部分得分 + DR平均分 + 需求分解与追踪质量得分
  - DR平均分 = 同一 OR 下所有 DR 得分的算术平均值
  - 整体报告总分 = 所有 OR 总得分的算术平均值
