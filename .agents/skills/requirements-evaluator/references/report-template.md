# 中文报告模板

按下面结构输出中文 Markdown 报告。

## 1. 评估概览

- 数据源
- `sheet_name`（如果输入是 Excel）
- OR 条目数
- DR 条目数
- 评分方法
- 总体平均分
  说明：总体平均分 = 所有 OR 总得分的算术平均值
- 等级分布
- 是否达到“优秀设计标准”的总体结论

## 2. 总体发现

- 最常见优点
- 最常见问题
- 最薄弱的维度
- 最值得优先修补的结构性问题

## 3. 单个 OR 评估结果

每个 OR 至少包含：

- OR 标识与名称
- OR 总得分与等级
- OR 部分得分
- 各 DR 分数
- DR 平均分
- 需求分解与追踪质量得分
- 关键证据
- 红旗问题
- 缺失项
- 修改建议

建议格式：

```md
### OR N: <编号> <名称>

- OR总得分: 78/100
- 等级: B
- 结论: 设计基础可用，但还未达到优秀设计标准
- OR部分得分: 30/40
- DR得分:
  - DR-1 `<编号> <名称>`: 34/40
  - DR-2 `<编号> <名称>`: 28/40
- DR平均分: 31/40
- 需求分解与追踪质量得分: 17/20

| 部分/维度 | 分数 | 说明 |
| --- | ---: | --- |
| OR-用户语言描述 | 10/12 | ... |
| OR-应用场景 | 8/12 | ... |
| OR-用户价值 | 7/10 | ... |
| OR-约束和限制 | 5/6 | ... |
| DR-1总分 | 34/40 | ... |
| DR-2总分 | 28/40 | ... |
| 需求分解与追踪-范围与边界 | 5/6 | ... |
| 需求分解与追踪-假设与依赖 | 4/6 | ... |
| 需求分解与追踪-一致性与可追踪性 | 4/4 | ... |
| 需求分解与追踪-异常与边界场景 | 4/4 | ... |

关键证据：
- ...

红旗问题：
- ...

缺失项：
- ...

修改建议：
1. ...
2. ...
3. ...
```

## 4. 优先级建议

按高、中、低列出跨 OR 的改进动作。

## 5. 附录

- 维度定义摘要
- 评分边界说明
- 分数计算说明：
  - 单个 OR 总分 = OR部分得分 + DR平均分 + 需求分解与追踪质量得分
  - DR平均分 = 同一 OR 下所有 DR 得分的算术平均值
  - 整体报告总分 = 所有 OR 总得分的算术平均值
